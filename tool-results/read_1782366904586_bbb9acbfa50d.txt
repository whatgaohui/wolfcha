    27тЖТ    27тЖТ## 1. LLM Integration
    28тЖТ    28тЖТ
    29тЖТ    29тЖТ### 1a. Server-side LLM proxy тАФ `src/app/api/chat/route.ts` (979 lines)
    30тЖТ    30тЖТ
    31тЖТ    31тЖТThis is the single gateway every LLM call goes through. It is a Next.js `POST` route handler that proxies OpenAI-compatible `/chat/completions` requests to **3 providers** based on the model id (looked up via `ALL_MODELS` / `PROJECT_MODELS`).
    32тЖТ    32тЖТ
    33тЖТ    33тЖТ**Providers and endpoints:**
    34тЖТ    34тЖТ| Provider | Type | Base URL |
    35тЖТ    35тЖТ|---|---|---|
    36тЖТ    36тЖТ| `zenmux` | OpenAI-compatible chat | `https://zenmux.ai/api/v1/chat/completions` |
    37тЖТ    37тЖТ| `dashscope` | Aliyun Bailian OpenAI-compatible mode | `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions` |
    38тЖТ    38тЖТ| `tokendance` | OpenAI-compatible gateway | `${TOKENDANCE_BASE_URL}/chat/completions` (default `https://tokendance.agent-universe.cn/gateway/v1`) |
    39тЖТ    39тЖТ
    40тЖТ    40тЖТ> Note: NewAPI is not present despite the task description mentioning it. Only ZenMux/DashScope/TokenDance are wired up.
    41тЖТ    41тЖТ
    42тЖТ    42тЖТ**Routing logic** (`getProviderForModel(model)`):
    43тЖТ    43тЖТ- Looks up the model id in `ALL_MODELS` first, then `PROJECT_MODELS`, and uses the `provider` field. If neither matches, returns `null` and the request is rejected with 400.
    44тЖТ    44тЖТ- The client can also force a provider via `body.provider` ("zenmux" | "dashscope" | "tokendance").
    45тЖТ    45тЖТ
    46тЖТ    46тЖТ**Auth model тАФ dual mode (server key vs. user-supplied key):**
    47тЖТ    47тЖТ- If the request includes any of the headers `x-zenmux-api-key`, `x-dashscope-api-key`, `x-tokendance-api-key`, it's treated as "custom-key mode" and that header value overrides `process.env.*`.
    48тЖТ    48тЖТ- `hasAnyCustomKeyHeader` gates fallback: if any custom key header is present but the required one for the chosen provider is missing, request is rejected with 401 (no silent fallback to server key).
    49тЖТ    49тЖТ- Built-in (no custom key) requires `requireCredits(auth.user.id)` or `hasRecentUnfinishedGameSession`.
    50тЖТ    50тЖТ
    51тЖТ    51тЖТ**Request body shape (`ChatRequestPayload`, lines 280тАУ290):**
    52тЖТ    52тЖТ```ts
    53тЖТ    53тЖТ{
    54тЖТ    54тЖТ  model: string;
    55тЖТ    55тЖТ  messages: unknown[];          // OpenAI-style {role, content} array; content may be string or multipart array
    56тЖТ    56тЖТ  temperature?: number;          // default 0.7; capped to 0..1 for ZenMux/Moonshot/Kimi
    57тЖТ    57тЖТ  max_tokens?: number;           // min 16
    58тЖТ    58тЖТ  stream?: boolean;
    59тЖТ    59тЖТ  reasoning?: { enabled: boolean; effort?: "minimal"|"low"|"medium"|"high"; max_tokens?: number };
    60тЖТ    60тЖТ  reasoning_effort?: "minimal"|"low"|"medium"|"high";
    61тЖТ    61тЖТ  response_format?: unknown;     // { type: "json_object" } | { type: "json_schema", json_schema: {...} }
    62тЖТ    62тЖТ  provider?: "zenmux" | "dashscope" | "tokendance";
    63тЖТ    63тЖТ}
    64тЖТ    64тЖТ```
    65тЖТ    65тЖТ
    66тЖТ    66тЖТ**Batch mode:** If the body has `{ requests: [...] }`, the route runs `runBatchItem()` for each item via `Promise.all` and returns `{ results: [...] }`. Batch mode **does not support streaming** (returns 400 if `stream=true`).
    67тЖТ    67тЖТ
    68тЖТ    68тЖТ**Streaming format:** SSE тАФ `Content-Type: text/event-stream`, `Cache-Control: no-cache`, `Connection: keep-alive`. The route just forwards `response.body` raw from the upstream provider (it does NOT re-frame the chunks). Terminator is `data: [DONE]`. Each line is `data: {json}` where `json.choices[0].delta.content` carries the text delta.
    69тЖТ    69тЖТ
    70тЖТ    70тЖТ**Per-provider transformations the route performs (important for z.ai migration):**
    71тЖТ    71тЖТ- `flattenMultipartContent` тАФ for models that don't support multipart content (only `z-ai/glm`, `volcengine/doubao` per `supportsMultipartContent`), arrays are joined into a single string.
    72тЖТ    72тЖТ- `coalesceTextOnlyMultipartContent` + `prependDeepSeekStablePrefix` тАФ DeepSeek-specific prefix-caching optimization. Inserts a `WOLFCHA_DEEPSEEK_CACHE_PREFIX_V1` marker system message to improve DeepSeek's automatic prefix cache hits.
    73тЖТ    73тЖТ- `stripCacheControl` тАФ removes `cache_control` field from content parts for models that don't support explicit caching (everything except `anthropic/*` and `qwen/*`).
    74тЖТ    74тЖТ- `withDashscopeJsonHint` тАФ DashScope: if response_format is json_object but messages don't mention "json", prepend `{role:"system", content:"Respond in json."}`.
    75тЖТ    75тЖТ- `normalizeDashscopeModelName` тАФ strips `qwen/` prefix for DashScope (since DashScope uses raw `qwen-*` names).
    76тЖТ    76тЖТ- `toZenMuxReasoning` тАФ ZenMux accepts `{enabled, effort, max_tokens}` only (no `exclude`).
    77тЖТ    77тЖТ- `toTokendanceThinking` тАФ TokenDance uses `{type:"enabled"|"disabled", budget_tokens:number}`. GLM/Kimi models default to thinking ON, so the route forces `{type:"disabled"}` unless explicitly enabled.
    78тЖТ    78тЖТ
    79тЖТ    79тЖТ**Provider-specific request bodies:**
    80тЖТ    80тЖТ- **ZenMux** (lines 905тАУ947): `{model, messages, temperature, max_tokens?, stream?, reasoning: {enabled:false}|..., response_format? (only if model supports it)}`. Headers: `Authorization: Bearer <key>`, `Content-Type: application/json`.
    81тЖТ    81тЖТ- **DashScope** (lines 721тАУ752): `{model (normalized), messages, temperature, max_tokens?, stream?, response_format?}`. Headers: `Authorization: Bearer <key>`, `Content-Type: application/json`. No `reasoning` field.
    82тЖТ    82тЖТ- **TokenDance** (lines 813тАУ855): `{model, messages, temperature, max_tokens?, stream?, thinking?, response_format? (only if model supports it)}`. Headers: `Authorization: Bearer <key>`, `Content-Type: application/json`, `X-App-Name: Wolfcha`, `X-Site-URL: https://wolf-cha.com`.
    83тЖТ    83тЖТ
    84тЖТ    84тЖТ**Other details:**
    85тЖТ    85тЖТ- Global undici dispatcher configured at module load with 60s connect timeout (default 10s) to handle slow Chinese API gateways.
    86тЖТ    86тЖТ- `API_TIMEOUT_MS = 60000` enforced via `AbortController` per upstream call.
    87тЖТ    87тЖТ- `supportsResponseFormat(model)` тАФ only `openai/`, `google/`, `anthropic/`, `deepseek/`, `qwen/`, `moonshotai/` prefixes. z-ai/glm and others are skipped.

   105тЖТ   105тЖТ  role: "system" | "user" | "assistant";
   106тЖТ   106тЖТ  content: string | LLMContentPart[];

   108тЖТ   108тЖТ}
   109тЖТ   109тЖТ
   110тЖТ   110тЖТinterface GenerateOptions {
   111тЖТ   111тЖТ  model: string;
   112тЖТ   112тЖТ  provider?: Provider;
   113тЖТ   113тЖТ  messages: LLMMessage[];
   114тЖТ   114тЖТ  temperature?: number;
   115тЖТ   115тЖТ  max_tokens?: number;
   116тЖТ   116тЖТ  reasoning?: ReasoningOptions;
   117тЖТ   117тЖТ  reasoning_effort?: "minimal" | "low" | "medium" | "high";
   118тЖТ   118тЖТ  response_format?: ResponseFormat;

   120тЖТ   120тЖТ
   121тЖТ   121тЖТinterface ChatCompletionResponse {  // OpenAI-shaped
   122тЖТ   122тЖТ  id: string;
   123тЖТ   123тЖТ  choices: { message: { role: "assistant"; content: string; reasoning_details?: unknown }; finish_reason: string }[];
   124тЖТ   124тЖТ  usage?: { prompt_tokens; completion_tokens; total_tokens; prompt_cache_hit_tokens?; prompt_cache_miss_tokens?; prompt_tokens_details?: { cached_tokens? } | null };
   125тЖТ   125тЖТ}
   126тЖТ   126тЖТ```
   127тЖТ   127тЖТ
   128тЖТ   128тЖТ**Exported functions:**
   129тЖТ   129тЖТ1. `generateCompletion(options): Promise<{ content, reasoning_details?, raw }>` тАФ non-streaming. POSTs to `/api/chat`, returns the first choice's content with `<think>` blocks stripped via `stripReasoningArtifacts`.
   130тЖТ   130тЖТ2. `generateCompletionBatch(requests): Promise<BatchCompletionResult[]>` тАФ POSTs `{requests: [...]}` to `/api/chat`, returns one result per item.
   131тЖТ   131тЖТ3. `async function* generateCompletionStream(options): AsyncGenerator<string>` тАФ POSTs with `stream:true`, parses SSE `data: ` lines, yields each `choices[0].delta.content` string. Has a state machine for stripping `<think>...</think>` blocks mid-stream. Skips `data: [DONE]`.
   132тЖТ   132тЖТ4. `generateJSON<T>(options & { schema?: string }): Promise<T>` тАФ wraps `generateCompletion`, appends "Respond with valid JSON only..." suffix to the last user message, forces `response_format: {type:"json_object"}` for ZenMux models if not set, parses with `parseJsonTolerant` (handles markdown fences, smart quotes, trailing commas, dangling quotes). On parse failure, retries once with a corrective user turn.
   133тЖТ   133тЖТ5. `mergeOptionsFromModelRef(modelRef, options)` тАФ applies `temperature` / `reasoning` / `provider` overrides from a `ModelRef`.
   134тЖТ   134тЖТ6. `resolveApiKeySource(model): "user" | "project"` тАФ checks `isCustomKeyEnabled()` and per-provider key presence.
   135тЖТ   135тЖТ7. `stripReasoningArtifacts(text)` and `stripMarkdownCodeFences(text)` тАФ public utilities.
   136тЖТ   136тЖТ8. `isQuotaExhaustedMessage(message)` тАФ checks for the `[QUOTA_EXHAUSTED]` marker.
   137тЖТ   137тЖТ9. `extractPromptCacheUsage(usage)` тАФ normalizes prompt-cache stats from various provider formats.
   138тЖТ   138тЖТ
   139тЖТ   139тЖТ**Key behavior:** When `isCustomKeyEnabled()` is `false`, `resolveModelForBuiltin(model)` rewrites any non-`PROJECT_MODELS` model to the first `AVAILABLE_MODELS` entry (currently `tokendance/deepseek-v4-pro`). This means: if the user's saved model isn't in the built-in project pool, it silently falls back to `deepseek-v4-pro`.
   140тЖТ   140тЖТ
   141тЖТ   141тЖТ**Retry policy:** `fetchWithRetry(input, init, maxAttempts=4)` with retryable statuses `{429, 500, 502, 503, 504}`, exponential backoff with jitter, honors `Retry-After` header (capped at 15s).
   142тЖТ   142тЖТ
   143тЖТ   143тЖТ**Custom-key headers built by `buildCustomKeyHeaders` (only when `isCustomKeyEnabled()` returns true):**
   144тЖТ   144тЖТ```
   145тЖТ   145тЖТX-Zenmux-Api-Key, X-Dashscope-Api-Key, X-Tokendance-Api-Key, X-Tokendance-Base-Url
   146тЖТ   146тЖТ```

   253тЖТ   253тЖТ## 2. TTS Integration
   254тЖТ   254тЖТ
   255тЖТ   255тЖТ### 2a. Server route тАФ `src/app/api/tts/route.ts` (400 lines)
   256тЖТ   256тЖТ
   257тЖТ   257тЖТNext.js `POST` handler, `runtime = "nodejs"`, `dynamic = "force-dynamic"`.
   258тЖТ   258тЖТ
   259тЖТ   259тЖТ**Provider:** MiniMax T2A V2 only.
   260тЖТ   260тЖТ
   261тЖТ   261тЖТ**Request body:** `{ text: string, voiceId: string }` (JSON).
   262тЖТ   262тЖТ
   263тЖТ   263тЖТ**Response:** binary audio bytes with a sniffed `Content-Type` (`audio/mpeg`, `audio/wav`, or `audio/ogg`). On error returns JSON `{error, ...}`. Adds `X-Minimax-Voice-Id-Requested` and `X-Minimax-Voice-Id-Used` headers (the latter may differ if a fallback voice was used).
   264тЖТ   264тЖТ
   265тЖТ   265тЖТ**Auth:**
   266тЖТ   266тЖТ- Headers `x-minimax-api-key`, `x-minimax-group-id` (custom-key mode) override env vars.
   267тЖТ   267тЖТ- Env vars: `MINIMAX_API_KEY`, `MINIMAX_GROUP_ID`, `MINIMAX_API_BASE_URL` (default `https://api.minimax.chat`), `MINIMAX_TTS_MODEL` (default `speech-01-turbo`).
   268тЖТ   268тЖТ- If no custom key header: requires `requireCredits(auth.user.id)`.
   269тЖТ   269тЖТ
   270тЖТ   270тЖТ**Upstream request payload (lines 61тАУ77):**
   271тЖТ   271тЖТ```json
   272тЖТ   272тЖТ{
   273тЖТ   273тЖТ  "model": "speech-01-turbo",
   274тЖТ   274тЖТ  "text": "...",
   275тЖТ   275тЖТ  "stream": false,
   276тЖТ   276тЖТ  "voice_setting": { "voice_id": "...", "speed": 1.0, "vol": 1.0, "pitch": 0 },
   277тЖТ   277тЖТ  "audio_setting": { "sample_rate": 32000, "bitrate": 128000, "format": "mp3", "channel": 1 }
   278тЖТ   278тЖТ}
   279тЖТ   279тЖТ```
   280тЖТ   280тЖТ
   281тЖТ   281тЖТ**URL:** `${baseUrl}/v1/t2a_v2?GroupId=${groupId}`. Falls back between `api.minimax.chat` and `api.minimaxi.com` automatically if the primary fails. Uses raw `node:https` (not `fetch`) so it can handle gzip/br/deflate decompression manually.
   282тЖТ   282тЖТ
   283тЖТ   283тЖТ**Audio extraction logic (lines 268тАУ378):** MiniMax may return:
   284тЖТ   284тЖТ1. Direct binary audio (forwarded as-is).
   285тЖТ   285тЖТ2. JSON with `data` field containing base64 or hex encoded audio (decoded).
   286тЖТ   286тЖТ3. JSON with `audio.url` or `data.url` (HTTP-fetched separately).
   287тЖТ   287тЖТ4. JSON with `base_resp.status_code != 0` тЖТ error. Code `2054` triggers a voice-id fallback to `DEFAULT_VOICE_ID` (male/female by gender hint) and one retry.
